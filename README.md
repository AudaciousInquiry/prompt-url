# prompt-url

**The `prompt:` URL scheme for AI agent session provenance.**

A `prompt:` URL is a stable, human-readable identifier for a specific user prompt within an AI agent session. It encodes the agent identity, session identifier, and timestamp (or internal prompt ID) needed to locate the originating prompt in the agent's session log.

## Example

```
prompt://claude-code/34d129b9-9dad-439d-92fa-8b9d2d3adb77/~2e150465-a33e-4354-929c-fa2347c8bd00
prompt://github-copilot/95602ecf-b509-42b5-b5c4-cefb007525df/2026-07-19T08:34:49.255Z
```

## Why "URL" and not "URI"?

A `prompt:` URL is a **locator** — it points to a resolvable resource (the prompt text and context in a session log). It is therefore more precisely a URL than a bare URI. However, the IETF registers schemes under the "URI Schemes" registry (RFC 7595), so the formal IETF submission uses "URI scheme" terminology. In everyday use, "prompt URL" is the preferred term.

## What It Solves

AI agents produce artifacts — code changes, documents, analysis outputs — whose provenance is difficult to trace. Session IDs identify a conversation but not which turn produced a given artifact. Timestamps are available but don't encode the agent or session. Internal prompt IDs aren't exposed to tool callers at call time.

A `prompt:` URL combines all three into a single embeddable reference, suitable for:

- **Git commit trailers** — `Prompt-URL: prompt://claude-code/...`
- **File metadata** — YAML front matter, SKILL.md update records
- **Audit trails** — cross-agent provenance chains
- **MCP tool provenance** — stamped into any file written by an MCP tool

## Syntax

```
prompt://<agent-name>/<session-ref>/<prompt-ref>

agent-name   = stable short name (e.g. claude-code, github-copilot)
session-ref  = UUID session identifier, or a registered human-readable alias
prompt-ref   = RFC 3339 timestamp (fallback), or ~<internal-prompt-id> (preferred)
```

The `~`-prefixed form is canonical when the agent exposes an internal prompt correlation ID.

## IETF Status

`draft-boone-prompt-uri-scheme-00` is an individual Internet-Draft targeting the
[agentproto BoF](https://datatracker.ietf.org/doc/bofreq-krishnan-agent-communication-protoco)
at IETF 126 (Vienna, July 2026). The scheme requests provisional registration in
the IANA URI Schemes registry under RFC 7595.

## Building the Draft

Requires [xml2rfc](https://github.com/ietf-tools/xml2rfc):

```
pip install xml2rfc
xml2rfc docs/draft-boone-prompt-uri-scheme-00.xml --text --html
```
